## Overview

<CompDetails name={componentName} />

The `ns-tabs` component separates related content into clearly labelled sections. It comprises multiple `ns-tab` components.

### Examples

<ExampleViewer type="standard" />

## Guidance

:::do
- Always add heading text.
- Use at least two tabs.
- Be mindful of the extra page weight.
:::

:::dont
- Add an icon to one tab without adding one to all other tabs.
- Add more than two words to the heading.
- Mix different types of [icons](/guidelines/assets#icons) (i.e. solid, outline, functional).
- Use an excessive amount of tabs.
:::

### Standard

![Labelled diagram of ns-tabs](@assets/components/ns-tabs/guidance.webp)

#### Key

| Key | Field type | Guidelines |
| :--- | :--- | :--- |
| <span class="marker">A</span> | Icon | Optional. Related to the label. [See our icons](/guidelines/assets/#icons). |
| <span class="marker">B</span> | Label | Use 1 or 2 words to infer the content of the associated [`<ns-panel>`](/compnents/ns-panel/). |
| <span class="marker">C</span> | Selected Tab | This is the visual state of a selected tab. Used to show the user which content is visible below. |
| <span class="marker">D</span> | Panel | This is the area in which the content for each [`<ns-tab>`](/components/ns-tab) will be displayed. |
