## Overview

<CompDetails name={componentName} />

Showcase a product, service or option which users can select. The `ns-selector` component can function as a single or multiple selection for the user.

:::updated
**27 March 2026**

Selector documentation now includes the `inform` and `lockup` variants.

Use `inform` when the supporting content should be understood before the user makes a selection. Use `type="lockup"` when the options need stronger hierarchy, richer imagery, and more emphasis.
:::

## Guidance

:::do
- Use an `image` or a `decoration` to support the label.
- Use an image size of [640 x 360px](/guidelines/assets#640x360px). Images should be WebP file type, the file size should be no more than 25kb.
- Use a heading within the content when providing supporting text.
- Use inline text links within supporting content if necessary.
- Use a `radio` button or `checkbox`.
- Use a [`<ns-expander>`](/components/ns-expander) if there's a need to include more details.
- Use a description when using a `decoration` (illustration).
- Use `inform` when the decision depends on the supporting content.
- Use `type="lockup"` when selectors need richer imagery and stronger visual hierarchy.
- Keep the same selector layout style across a related group of options.
:::

:::dont
- Mix `radio` with `checkbox` types.
- Use it if the `image` or `decoration` makes the selection more difficult to understand.
- Add too much content - keep things concise.
- Use images that contain embedded text.
- Use an `image` and a `decoration` together.
- Exceed five expanders on a `<ns-selector>`.
- Use `inform` when users need to scan and compare many options quickly.
- Mix `type="lockup"` with standard selectors in the same group.
- Use `keyline` with `type="lockup"`; use `scheme` instead.
- Put `type="lockup"` in a component that has padding, see [Lockup placement](#lockup-placement) for details.
:::

### Standard

![Labelled diagram of ns-selector standard](@assets/components/ns-selector/guidance-standard.webp)

#### Key

| Key | Field type | Guidelines |
| :--- | :--- | :--- |
| <span class="marker">A</span> | Label | Describe the selection concisely. It should not exceed 24 characters. |
| <span class="marker">B</span> | Heading | A heading provides further description to the label. The recommended length is between 4 and 12 words, not exceeding 50 characters in total. |
| <span class="marker">C</span> | Paragraph | A single, short paragraph followed by an unordered list works best. The text length should not exceed one paragraph and a ticked list of 3 to 5 items. Include emphasis with `<strong>`, inline links `<a>`, lists `<ul>` `<ol>`. Add a [caveat link](/components/ns-caveat) if required. In an `standard` selector, this content should support the value of the label. |
| <span class="marker">D</span> | Decoration | Use to support and highlight the content. [See illustrations](/guidelines/assets/#illustrations). |
| <span class="marker">E</span> | Keyline | An optional colour keyline can be added to distinguish and draw attention to standard and `inform` selectors. `keyline` is not supported on `type="lockup"`. |
| <span class="marker">F</span> | Pill | Use a pill to highlight something distinctive, such as a discount, promotion, or eco benefit. Place it above the heading and supporting content. Use it sparingly so it keeps its attention-grabbing impact. |
| <span class="marker">G</span> | Expanders | An optional [`<ns-expander>`](/components/ns-expander) can be added within the selector to reveal more detail. It contains a heading and anonymous content slot. When the heading is clicked the content opens, and clicking the heading again hides it. |

### Inform

![Labelled diagram of ns-selector inform](@assets/components/ns-selector/guidance-inform.webp)

Use the `inform` attribute when the content is informing the user of the option, not supporting information. Placing the form control beneath the description encourages users to read the benefits, caveats, or explanatory copy before selecting.

#### Key

| Key | Field type | Guidelines |
| :--- | :--- | :--- |
| <span class="marker">A</span> | Label | Describe the selection concisely. It should not exceed 24 characters. |
| <span class="marker">B</span> | Heading | A heading provides further description to the label. The recommended length is between 4 and 12 words, not exceeding 50 characters in total. |
| <span class="marker">C</span> | Paragraph | A single, short paragraph followed by an unordered list works best. The text length should not exceed one paragraph and a ticked list of 3 to 5 items. Include emphasis with `<strong>`, inline links `<a>`, lists `<ul>` `<ol>`. Add a [caveat link](/components/ns-caveat) if required. In an `inform` selector, this content should explain the value of the label before the user selects it. |

### Lockup

![Labelled diagram of ns-selector lockup](@assets/components/ns-selector/guidance-lockup.webp)

Use `type="lockup"` when the selector is used for marketing and offers. This layout works well when imagery and visual hierarchy are helping the user understand the offer.

#### Key

| Key | Field type | Guidelines |
| :--- | :--- | :--- |
| <span class="marker">A</span> | Label | Describe the selection concisely. It should not exceed 24 characters. |
| <span class="marker">B</span> | Heading | A heading provides further description to the label. The recommended length is between 4 and 12 words, not exceeding 50 characters in total. |
| <span class="marker">C</span> | Paragraph | A heading with a single, short paragraph followed by an unordered list works best. The text length should not exceed one paragraph and a ticked list of 3 to 5 items. Include emphasis with `<strong>`, inline links `<a>`, lists `<ul>` `<ol>`. Add a [caveat link](/components/ns-caveat) if required. |
| <span class="marker">D</span> | Image | Use an image size of [640 x 360px](/guidelines/assets#640x360px). Images should be WebP file type, the file size should be no more than 25kb. |
| <span class="marker">E</span> | Scheme | Use a `scheme` to distinguish and draw attention to `lockup` selectors. The scheme is applied to the whole selector |

## Implementation

### Placement

<Placement name={componentName} />

#### Lockup placement

Pay close attention to the placement of `type="lockup"`. The HTML markup **must** be:

```html
<ns-panel>
  <ns-form>
    <ns-inputter>
      <ns-selector type="lockup">
          <!-- content -->
      </ns-selector>
    </ns-inputter>
  </ns-form>
</ns-panel>
```
