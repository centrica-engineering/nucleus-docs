## Overview

<CompDetails name={componentName} />

Use the `ns-standout` component to make information stand out from a page and to visually differentiate a concise block of text from the content that surrounds it, for example:

- Quotes (as in prices)
- Call-outs
- Examples
- Additional information about the page
- Emergency information

## Guidance

:::do
- Keep concise and to the point.
- Limit use to 1 or 2 per page.
- Use it to make information stand out.
- Select a colour border decoration that works with your design.
- Use a relevant, single text link if it is necessary.
- Use without an `<h2>` - `<h6>`.
- Use without a link.
- Ensure the information is relevant and important enough to stand out from the rest of the content.
:::

:::dont
- Fill it with too many paragraphs.
- Overuse it - it dilutes the impact.
- Use it instead of [`<ns-alert>`](/components/ns-alert) or [`<ns-card>`](/components/ns-card) â€“ they have different purposes.
- Add icons, images or illustrations.
- Use the `direct` type [`<ns-cta>`](/components/ns-cta).
- Nest a `<ns-standout>` component within another.
- Use in a form where helper text is more appropriate.
- Use for warning or error messages.
:::

#### Key

| Key | Field type | Guidelines |
| :--- | :--- | :--- |
| <span class="marker">A</span> | Decoration | The colour of the border. See [`decoration`](#specification-attribute-decoration) for the list of options. |
| <span class="marker">B</span> | Content | The content can accommodate a heading and multiple paragraphs, these should be concise. Can contain bold copy `<b>`, inline links `<a>`, lists `<ul>` `<ol>`, and a caveat at the end of the relevant paragraph if required `<a href="#caveat">1</a>`.|
