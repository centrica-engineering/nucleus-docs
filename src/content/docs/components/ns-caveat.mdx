## Guidance

:::do
- Try to make the caveats positive.
- Be transparent â€“ if the legal details are short then add them to the content.
- Always have the legal caveats at the bottom of the page, before the footer.
- Use an ordered list for the caveats that are linked in the page.
- Use a `<p>` tag for the caveats that relate to the page or journey in general.
- Only use text in the caveats.
- Embolden text that is important.
:::

:::dont
- Use the caveat to hide information.
- Add images, iconography, etc.
- Try to change the font size of the caveats.
:::

#### Key

| Key | Field type | Guidelines |
| :--- | :--- | :--- |
| <span class="marker">A</span> | Link | Clicking the caveat link will navigate to [`<ns-caveat>`](/components/ns-caveat/) at the bottom of the page. |
| <span class="marker">B</span> | Heading | The recommended length is between 4 and 12 words, not exceeding 50 characters in total. |
| <span class="marker">C</span> | Anonymous slot | An ordered list `<ol>` is recommended to associate the number used in the caveat link. A paragraph `<p>` is for generic fine print and legal information. |
