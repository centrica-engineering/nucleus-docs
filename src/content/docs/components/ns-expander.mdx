## Overview

<CompDetails name={componentName} />

An expander consists of a heading and an anonymous content slot. Click on a heading to reveal the content. Click it again to hide the content.

Use an `ns-expander` component in an accordion, a product card, or other contexts.

## Guidance

:::do
- Use instead of a modal.
- Use headings, paragraphs, and lists for content.
- Relate the heading to the content.
- Use a coloured expander to draw attention to a promotion on a [`<ns-product-card>`](/components/ns-product-card).
:::

:::dont
- Use to hide important information.
- Use as a solution to excessive copy on a page.
- Nest expanders within expanders.
- Use multiple coloured expanders on one page.
- Use a coloured expander to draw attention to a warning message - use [`<ns-alert>`](/components/ns-alert) or [`<ns-skyline>`](/components/ns-skyline) instead.
:::

### Standard

#### Key

| Key | Field type | Guidelines |
| :--- | :--- | :--- |
| <span class="marker">A</span> | Heading | Introduces the content in the anonymous slot. Clicking the headings reveals the content. The recommended length is between 4 and 12 words, not exceeding 50 characters in total. |
| <span class="marker">B</span> | Anonymous slot | Include headings, lists and paragraphs. Divide content into short paragraphs as it is easier to consume, and aids effective page scanning. Include emphasis with `<strong>`, inline links `<a>`, lists `<ul>` `<ol>`. Add a [caveat link](/components/ns-caveat) if required. |
