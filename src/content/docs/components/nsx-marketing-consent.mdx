## Overview

<CompDetails name={componentName} />

Use this experience pattern in any journey where there is a need to capture and submit customer marketing preferences. Only use it for this purpose – do not use it for viewing or changing existing preferences.

:::updated
**xx March 2026**

### Upgrade

A new `version` attribute is available which toggles the following: 

- new content and checkbox layout
- `has-opt-out` attribute will toggle an opt-out checkbox instead of a link
- `is-required` attribute toggles validation

Check those options in [the specification](#specification) down below!
:::

## Guidance

:::do
- Always keep the same options.
- Use at the end of a journey to capture contact information for marketing purposes.
- Always use within an [`<ns-form>`](/components/ns-form) component.
- Use only once per journey.
:::

:::dont
- Reorder the options.
- Use for collecting contact information solely for offering a service, e.g. Booking an engineer.
- Have any of the options pre-checked.
:::

### Standard

![Labelled diagram of nsx-marketing-consent](@assets/components/nsx-marketing-consent/guidance.webp)

It is possible to change the heading and the first paragraph to suit specific context. However, we recommend using the default content as these have been signed off by the legal team.

If there is an official update to the default content, please inform the Nucleus team so that the the component can be updated.
