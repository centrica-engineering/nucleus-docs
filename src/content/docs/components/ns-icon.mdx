## Overview

<CompDetails name={componentName} />

Icons provide clarity to users and reduce their cognitive load. Icons always supplement a single line of text and are not treated as a description. They are solid or outline a single colour.

### Examples

All available icons can be seen on the [Icons](/guidelines/assets#icons) section of the Assets page.

<ExampleViewer type="standard" />

## Guidance

:::do
- Use icons within the component.
- Use this when building new Nucleus components.
- Use them in colours.
- Keep them small.
:::

:::dont
- Create new icons without following the proposal process.
- Add icons as content.
- Use a random mix of outline and solid icons.
:::
